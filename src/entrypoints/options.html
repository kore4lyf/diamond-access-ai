Diagnostics

Last 500 events from the service worker logger. Useful for development — click Refresh logs after using Diamond to inspect what it did: transcripts (with the user’s voice text), dispatched actions, LLM round-trips, TTS utterances, errors, sleep-timer resets, lifecycle events.
Privacy: sensitive field values are never logged — only the field type (“password”, “credit-card”, “ssn”, “cvv”, “dob”). LLM prompt and response bodies are also never logged. 


    


    

      Verbose LLM response log
      

        When diamond_verbose_llm=true, each LLM response body is captured here (last 50, capped). Useful for diagnosing what the model actually emitted vs. what the prompt expected. Survives Chromium restarts.
        

        Enable from SW DevTools console:
        await chrome.storage.local.set({ diamond_verbose_llm: true })
        

        Privacy: this panel only shows anything when you set the opt-in key. Real users never see a body here.
